UniqueJS30

What is UniqueJS30?

Hey there everyone, my name is David Nugent. I'm a developer based out of BC, Canada. At the end of July 2021 I decided to take on a project to develop an app a day for 30 days.

This was in-part inspired by Wes Bos' JS30 free course - which I would highly recommend for any junior dev's, you can find it here. In this course, the idea is to learn by building and to only use vanillaJS with no packages or frameworks. As an avid user of frameworks such as React I had some trepidation and looking at the course at a glance I wasn't too sure I would actually learn any new skills, so I decided to take it one step further by building upon the daily concepts and build something unique every day instead. Thus UniqueJS30 was born.

Below, as well as videos and descriptions, you can find live demos and source code. The sites below were originally built for desktop viewing but I will make these designs adaptive at a later stage. If you enjoy the concept or any of the projects, feel free to connect with me directly here.

Piano Emulator ⛛

For day one, I created a piano emulator. The piano emulator uses keycode detection on keyboard presses and maps these to specific audio elements which are subsequently played allowing a user to emulate the experience of pressing a key on a piano. In addition I added some dynamic transition effects to give the visual appearance of key presses.

12

David Says ⛛

For day twelve I created a reimagining of the classic electric game of Simon Says, dubbed 'David Says'. The idea for this game was inspired by Key Sequence Detection where user actions are logged and compared to a set criteria to illicit a specific response. I enjoyed exploring different methods for getting sequential audio clips to wait for one another, both creative use of an incremented setTimeOut inside a forEach and also by using Async/Await and Promises. I got my hands dirty with some audio manipulation as well - all of the voice-overs are actually me with the pitch edited.

13

Scroll Hijacking ⛛

For day thirteen I explored the topic of Scroll Hijacking.Scroll hijacking gets a bad rep for being poor for user accessibility and for hampering the functionality of a site, while I agree to a degree I also believe if executed well they can also add emphasis to the point of a site/page if they don't take functionality away from a user - this was one of the goals of this project. I enjoyed creating a debounce function to limit the number of event fires.

15

CHRONOS ⛛

For day fifteen, of my UniqueJS30DayChallenge I explored the use of localStorage to persist state/data across refreshes/re-navigation/tab close. To do this I applied logic to a timer app which tracks the current progress of a given task to a scope of a larger project. The logic behind the scenes for this app was quite complex in parts. I created a complex factory function to give scope/control to the individual timers and improve re-usability. I created a drift control for our timers to improve accuracy. I found a method for re-applying methods to the factory functions when restored from local storage. All of this was very fun and would encourage you to check out the code in the repo!

JavaScript W[arr]ior 3 ⛛

For day seventeen, I created a game (inspired in-part by some of the retro Final Fantasy games) where you explore a dungeon and get some random fight encounters. These encounters use a turn-based style of fighting. The main character and every monster have their own special attack with animations and sound effects as well. I explored some creative use of getBoundingClientRect() in order to map out blocked coordinates for where our character can move which allows greater flexibility and adds the ability to easily adjust maps. I created all of the sprites for the map and the attack animations myself which was time-consuming but fun!
